# Subscriptions Sync Dashboard (Backend)

## Что сделано?

- ```SubscriptionsController``` с двумя экшнами для получения списка подписок и постановки задачи на синхронизацию;
- Настройки пагинации и фильтрации передаются через query-параметры;
- Синхронизация реализована очередью на RabbitMQ - постраничное чтение из API Stripe (с помощью SDK) с последующим upsert в MySQL;
- Запросы на синхронизацию со статусами хранятся в таблице `subscriptions_sync_processes`.

## Комментарии (как можно улучшить)

- Текущая реализация будет работать ок для небольшого количества подписок, но для масштабирования нужно применять другой подход. Я это продумал так: использование вебхуков и фоновая cron-задача, которая выполняется раз в 5 минут и забирает существующие подписки по условию `last_synced_at < NOW() - INTERVAL 3 DAY` по 1000 штук (такого поля сейчас нет, его нужно добавить). Полную синхронизацию стоит оставить для самых крайних случаев;
- Добавить больше полей из Stripe - сейчас их очень мало;
- Не используются мертвые очереди - для экономии времени я пользовался готовым пакетом, в котором они не создаются.
